在這裡會紀錄大三上的資料結構與演算法課程中所有的上課內容，回家自己做的筆記以及演算法的各種原理與程式碼，還有課堂作業、Codesignal、Leetcode的解題與學習過程。
#### 什麼是演算法？
我在網路上看到了形容演算法的簡單定義，有點像是輸入 + 演算法 = 輸出，我輸入一個東西，想要得到另外一個東西，經過的過程就是所謂的演算法。（假設我們輸入2 還有3，想要得到 6，我們需要在中間加上一個演算法「乘號」）
 * 演算法相當於「思考程序」
 * 程式相當於為了表現演算法的「語言」

# 期末心得
資料結構與演算法這堂課絕對是大學三年以來最累人也最震撼的一堂課，不知道是不是因為之前兩年過得太輕鬆才導致這半年過得如此艱辛，其實在第一堂課就大概有感受到這會是一堂偏硬的課，直到交了第一次作業後才確定真的是一堂硬到爆的課....，最崩潰的時候應該是在寫第二次跟第三次作業的時候吧，因為我的程式語言基礎不好（非常不好），所以寫的程式碼很多地方都是充滿錯誤，而且這也不是一時之間就可以補上來的，花的時間遠遠的比起平常其他科還多上了許多，我絕對是認真用心想寫出來作業的，然而努力跟成果一點都不成正比，直接變成演算法時差每天2點前都沒可能睡覺，甚至連睡覺做夢都在寫程式碼（我真的不浮誇真的是這樣＾＾），然後一起床就會繼續馬上想到昨天還沒成功的部分，整個煩躁到不行，真的一度很想直接放棄，直接去退選，有時候有問題就算去問同學了，可能也聽不懂他的回答，因為真的沒什麼程式知識...，最常遇到的是我甚至根本連自己錯在哪裡都不知道，根本不知道該怎麼問，真的謝謝我的腦袋：），而在繳交第三次作業後，我覺得我某個心態上有了轉變，也是因為這個轉變所以我後來沒有過得那麼高壓那麼崩潰，好啦也應該是因為第三次作業後真的有變比較簡單了，雖然我前期真的過著很怨天尤人，但後期其實有發現到自己慢慢有在進步，而且真的是差蠻多的，多了很多程式基本的概念和想法，邏輯比較清楚了，腦袋思路也順了很多，也可以花比較短的時間去理解不同的程式碼，有時候甚至能幫助朋友解決一些簡單的問題，重新回去看之前寫的作業時，當初一直卡住的地方現在也有辦法解決了，也會發現一些當初沒發現的問題，改進它或是理解它，花費的時間開始跟成果有那麼一點點的正比，成就感當然是有的，之前一直埋怨自己為什麼理解能力那麼差，為什麼就是想不到解決辦法，現在終於能稍微打出一些些程式碼了，就算還沒辦法全部自己想出來，還是需要看參考資料，但我真的覺得已經比一開始好上許多了，我知道比起很多人我這樣還是遠遠差上很多，但我真的盡我所能的努力了！雖然在寫第六次作業時，又感受到之前的那種挫敗感，也有可能是因為同時要讀期末考所以有點分身乏術，但後來考完期末考後還是有成功交出作業，只能說真的是痛哭流涕。

做完這學期最後一次作業和考完期末考後，我真的鬆了很大口氣，感覺很多壓力都釋放了，這半年這樣看來，這堂課的確逼迫著我，讓我改變了很多，我指的不只是程式碼還有很多心態的部分，先謝謝我自己沒有放棄，一直堅持到了現在，然後雖然整學期都在超級心態炸裂的狀況下，但就某些角度來看，我還是要感謝這堂課，啊但我還是不想再來一次：），真的是祈求拜託讓我過了.......，謝謝....。
